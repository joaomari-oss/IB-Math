Módulo 1

Algoritmos (Algorithms)

A base de toda programação: sequências lógicas de instruções para resolver problemas

Conceito

O que é um Algoritmo?

Um algoritmo é uma sequência finita e ordenada de instruções bem definidas que, quando executadas, resolvem um problema ou realizam uma tarefa. É como uma receita de cozinha: cada passo precisa ser claro e executável.

💡
Analogia: Pense em como você prepara um café. Os passos são: (1) ferver a água, (2) colocar o pó no filtro, (3) despejar a água, (4) esperar filtrar, (5) servir. Essa sequência é um algoritmo!
Entrada (Input)

Dados que o algoritmo recebe para processar

Processamento

A lógica aplicada sobre os dados de entrada

Saída (Output)

O resultado produzido pelo algoritmo

Pseudocódigo — Representando Algoritmos

Pseudocódigo é uma forma de escrever algoritmos em linguagem natural estruturada, sem se prender à sintaxe de uma linguagem específica.

Pseudocódigo
ALGORITMO: Verificar se número é par
INÍCIO
  RECEBER número
  SE número MOD 2 == 0 ENTÃO
    EXIBIR "O número é par"
  SENÃO
    EXIBIR "O número é ímpar"
  FIM SE
FIM

Complexidade de Algoritmos

A eficiência de um algoritmo é medida pela sua complexidade de tempo (quanto tempo leva) e complexidade de espaço (quanta memória usa). Usamos a notação Big O para representar isso:

O(1) — Constante

Executar em tempo fixo, independente do tamanho dos dados

O(n) — Linear

Tempo cresce proporcionalmente ao tamanho dos dados

O(n²) — Quadrática

Tempo cresce com o quadrado dos dados (loops aninhados)

Caso Real

O Google processa bilhões de buscas por dia. Se usasse um algoritmo O(n²), demoraria horas. Com algoritmos O(log n) como busca binária, responde em milissegundos. A escolha do algoritmo correto é crucial!

Variáveis (Variables)

Contêineres nomeados que armazenam dados na memória do computador

Conceito

Declaração e Atribuição

Uma variável é um espaço na memória do computador identificado por um nome, onde armazenamos valores que podem mudar durante a execução do programa.

💡
Analogia: Pense em variáveis como caixas etiquetadas. A etiqueta é o nome (idade), e dentro da caixa colocamos o valor (16). Podemos trocar o conteúdo a qualquer momento.

Em JavaScript, existem três formas de declarar variáveis:

Escopo de Variáveis

O escopo determina onde uma variável pode ser acessada no código. Em JavaScript, existem dois tipos principais:

Escopo de Bloco (let/const)

A variável só existe dentro do bloco { } onde foi declarada. Mais seguro e previsível.

Escopo de Função (var)

A variável existe dentro de toda a função, mesmo antes de ser declarada (hoisting). Pode causar bugs.

Módulo 5

Estruturas de Repetição (Loops)

Executar blocos de código repetidamente enquanto uma condição for verdadeira

Conceito

Os Três Tipos de Loop

Loops permitem repetir instruções sem escrever o mesmo código várias vezes. Em JavaScript, os três loops principais são:

O loop for tem três partes: inicialização, condição e incremento.

for (let i = 0; i < 5; i++) {
  console.log("Iteração " + i);
}

O while repete enquanto a condição for verdadeira. Cuidado com loops infinitos!

let contador = 0;
while (contador < 3) {
  console.log("Contagem: " + contador);
  contador++;
}

Semelhante ao while, mas verifica a condição depois de executar.

let num = 10;
do {
  console.log("Número: " + num);
  num--;
} while (num > 7);
💡
Em jogos: O game loop é o loop mais importante da programação de jogos. Ele roda ~60 vezes por segundo, atualizando posições, verificando colisões e redesenhando a tela. Sem loops, não existem jogos!

Lógica Booleana

A álgebra booleana, criada por George Boole, trabalha com apenas dois valores: true (verdadeiro) e false (falso). É a base de toda a computação digital — os bits 0 e 1.

AND (&&)

Verdadeiro SOMENTE se AMBOS forem verdadeiros

OR (||)

Verdadeiro se PELO MENOS UM for verdadeiro

NOT (!)

Inverte o valor: true vira false e vice-versa

Módulo 7

Controle & Colisão (Control & Collision)

Como programar movimentação e detecção de colisões em jogos 2D

Conceito

Detecção de Colisão AABB

A colisão AABB (Axis-Aligned Bounding Box) é o método mais simples e eficiente para detectar se dois retângulos se sobrepõem. Funciona verificando se os limites de dois objetos se interceptam em ambos os eixos (X e Y).

Condição de Colisão AABB
A.x < B.x + B.w && A.x + A.w > B.x && A.y < B.y + B.h && A.y + A.h > B.y

Se TODAS as condições forem true, os retângulos estão colidindo.

Física de Jogo (Game Physics)

Simular gravidade, velocidade e aceleração para criar movimentos realistas

Conceito

Cinemática em Jogos 2D

A física de jogos simula leis do mundo real de forma simplificada. Os conceitos fundamentais são posição, velocidade e aceleração.

Equações do Movimento
posição += velocidade × dt
velocidade += aceleração × dt

dt (delta time) é o tempo entre frames, garantindo movimento consistente em diferentes FPS.

Módulo 9

Efeito Parallax (Parallax Effect)

Criar ilusão de profundidade movendo camadas em velocidades diferentes

Conceito

Como Funciona o Parallax

O efeito parallax cria uma ilusão de profundidade 3D em cenários 2D. Objetos distantes se movem mais devagar que objetos próximos — exatamente como na vida real quando você olha pela janela de um carro.

💡
Analogia: Quando você anda de carro, as árvores na beira da estrada passam rápido, as montanhas ao fundo se movem lentamente, e o sol/lua parece parado. Isso é parallax!
Jogos Famosos

Parallax scrolling é usado em jogos como Hollow Knight, Ori and the Blind Forest, Celeste e Limbo. Cada um usa múltiplas camadas (3 a 7 camadas) para criar cenários imersivos.

Módulo 11

Arquitetura de Jogos (Game Architecture)

O Game Loop, gerenciamento de estados e organização de cenas

Conceito

O Game Loop

O Game Loop é o coração de qualquer jogo. É um loop infinito que executa três etapas a cada frame: processar entrada (input), atualizar lógica (update) e renderizar (draw).

Ciclo do Game Loop
Input → Update → Render → Repeat (60 FPS)

A cada 16.67ms (60 FPS) o ciclo se repete, criando a ilusão de movimento.

Módulo 12

Mapas & Tiled (Tilemaps)

Construir mundos de jogo usando grids de tiles (blocos)

Conceito

O que são Tilemaps?

Um tilemap é uma grade 2D onde cada célula contém um tile (bloco) — uma pequena imagem que representa um pedaço do cenário. Combinando tiles, criamos mapas enormes de forma eficiente.

💡
Exemplo clássico: Super Mario Bros usa tilemaps! O chão, os blocos de interrogação, os canos — tudo são tiles de 16x16 pixels organizados em uma grade.

Processamento de Input

Em jogos, o input é processado a cada frame. Precisamos rastrear quais teclas estão pressionadas (não apenas quando são apertadas) para movimentos suaves e contínuos.

Event-Based

Módulo 16

Engenharia de Software (Software Engineering)

Processos, metodologias e boas práticas para construir software profissional

Conceito

SDLC — Ciclo de Vida do Software

O SDLC (Software Development Life Cycle) define as etapas para criar software de forma organizada:

1
Requisitos — O que o software precisa fazer?
2
Design — Como será a arquitetura e interface?
3
Implementação — Escrever o código
4
Testes — Verificar se funciona corretamente
5
Deploy — Entregar/publicar o software
6
Manutenção — Correções e melhorias contínuas

Metodologias Ágeis

Waterfall (Cascata)

Cada fase sequencial. Funciona para projetos com requisitos fixos. Difícil voltar atrás.

Agile (Ágil)

Iterativo e incremental. Sprints de 2-4 semanas. Adaptável a mudanças. Mais usado hoje.

Design Patterns

Boas Práticas
  • DRY (Don't Repeat Yourself) — Evite duplicação de código
  • KISS (Keep It Simple, Stupid) — Mantenha simples
  • SOLID — 5 princípios de design orientado a objetos
  • Code Review — Revise código com colegas antes de integrar
  • Version Control — Use Git para rastrear mudanças
